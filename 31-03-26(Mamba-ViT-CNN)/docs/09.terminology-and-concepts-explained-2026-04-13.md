# PART 1: THE MODELS

## What is "Mamba+CBAM+P2Head"?

This is YOUR model — the one you built. It's a modified YOLO11m object detector with three changes:

### 1. CBAM (Convolutional Block Attention Module)
- **What:** A lightweight attention module that helps the model focus on "what" (channel attention) and "where" (spatial attention) in an image
- **Where in the model:** Replaces C2PSA at backbone layer 10
- **How it works:**
  1. **Channel attention:** Looks at all feature channels and learns which ones are important (e.g., edge channels, color channels, texture channels). Uses global average pooling + global max pooling → shared MLP → sigmoid gating
  2. **Spatial attention:** Looks at all spatial positions and learns where to focus (e.g., focus on center of person, ignore background). Uses channel-wise avg+max pooling → 7x7 conv → sigmoid gating
- **Effect:** Helps the model pay attention to person-like features and ignore irrelevant background. Reduces parameters by 2.3% vs baseline C2PSA
- **Parameters added:** None (actually -2.3% vs C2PSA)

### 2. P2 Head (Extra Small-Object Detection Head)
- **What:** An additional detection head that operates at stride=4 (160x160 feature map from 640x640 input)
- **Why:** Standard YOLO11m detects at P3 (stride 8), P4 (stride 16), P5 (stride 32). A sub-10px person becomes ~1.25 pixels at P3 — nearly impossible to detect. At P2 (stride 4), it becomes ~2.5 pixels — still small but detectable
- **Standard YOLO11m:** 3 detection scales — P3 (80x80), P4 (40x40), P5 (20x20)
- **Your model:** 4 detection scales — P2 (160x160), P3 (80x80), P4 (40x40), P5 (20x20)
- **Effect:** +2.04% mAP50-95 and best very-tiny recall among all ablation variants
- **Trade-off:** +28.2% GFLOPs (34.1 → 43.7) due to the extra high-resolution feature map

### 3. C3K2Mamba (Mamba SSM in the Neck)
- **What:** Replaces standard C3K2 bottleneck blocks in the FPN/PAN neck with Mamba-based State Space Model blocks
- **Where:** Layers 13, 16, 19, 22, 25 in the neck (6 locations)
- **How it works:**
  1. Input features are split into two halves via Conv1x1 + chunk
  2. One half passes through MambaBottleneck blocks that use **LocalWindowSSM** (Local Window State Space Model)
  3. LocalWindowSSM partitions features into small windows (4x4, 6x6, or 8x8 depending on channel count), flattens them into sequences, then runs a **bidirectional selective scan** (forward + backward)
  4. The selective scan processes each sequence like a recurrent model but with linear O(n) complexity instead of O(n^2) like attention
  5. Results are merged, gated, and projected back
- **Why Mamba instead of attention?** Attention (like in ViT/DETR) has O(n^2) complexity — too slow for high-res feature maps. Mamba's SSM has O(n) complexity — processes long sequences efficiently
- **Effect:** +1.2% mAP50-95 over CBAM+P2 alone, best overall mAP50 (0.877)
- **Parameters added:** Zero (19.592M same as CBAM+P2 — Mamba replaces existing bottleneck, doesn't add new params)
- **Latency added:** +6.9ms (44.3ms → 51.2ms)

### What is "No-Aug" Model?

"No-aug" means the model was trained with **standard/default augmentation only** — basic mosaic, horizontal flip, HSV jitter. It did NOT use copy-paste, mixup, vertical flip, or rotation.

This is your **best model** — the one that gives mAP50=0.877, mAP50-95=0.654. All SAHI and TTA experiments use this model's weights.

### What is the "Copy-Paste" Model?

The same Mamba+CBAM+P2 architecture, but retrained from scratch with aggressive augmentation:
- `copy_paste=0.5` — 50% of training images get objects copy-pasted from other images
- `mixup=0.15` — 15% of images are blended with another image
- `flipud=0.5` — 50% vertical flip
- `degrees=10.0` — up to 10 degrees rotation

This model performed **worse** than the no-aug model. It's a negative result.

# PART 2: INFERENCE-TIME TECHNIQUES

## SAHI (Slicing Aided Hyper Inference)

### The Problem SAHI Solves
Your model was trained on 640x640 images. When a person is only 6 pixels tall in the original image, at 640x640 it might be 3-4 pixels. That's almost invisible to the detector.

### How SAHI Works — Step by Step

**Without SAHI (standard inference):**
```
Original image (e.g., 640x640)
    → Feed entire image to model
    → Model outputs detections
    → Done
```
A 6px person is still 6px. Hard to detect.

**With SAHI (sliced inference):**
```
Original image (640x640)
    → Slice into overlapping tiles (e.g., 256x256 with 30% overlap)
    → Each tile is resized to 640x640 for the model (upscaled!)
    → Model detects on each tile separately
    → Merge all tile detections back to original coordinates
    → Also run full-image detection (perform_standard_pred=True)
    → Merge tile detections + full-image detections
    → Apply postprocessing to remove duplicates
    → Final detections
```

A 6px person in a 256x256 tile gets upscaled to ~15px in the 640x640 model input — much more detectable!

### SAHI Configuration Parameters Explained

**Slice size (e.g., 256x256):**
- How big each tile is, in pixels
- Smaller slices = more upscaling = better tiny object detection = but slower
- We tested 4 sizes: 256, 320, 512, 640

**Overlap ratio (e.g., 0.30):**
- How much neighboring tiles overlap
- 0.30 means 30% overlap — ensures objects at tile borders aren't cut in half
- Too low: objects split between tiles get missed
- Too high: too many tiles, very slow

**Confidence threshold (e.g., 0.15):**
- Minimum confidence for a detection to be kept
- Standard is 0.25, but we lowered to 0.15 because sub-10px objects naturally produce low confidence scores
- Lower = more detections (including false positives) but fewer missed humans

**perform_standard_pred=True:**
- **CRITICAL setting.** Runs full-image inference IN ADDITION to sliced inference
- Without this: the model only sees small tiles, and medium/large objects get cut across tiles → medium recall collapses
- With this: medium/large objects are detected normally from the full image, small objects are detected from tiles
- Our previous SAHI attempt WITHOUT this setting lost -13.3% medium recall. This time: 0% medium recall loss

**postprocess_type=GREEDYNMM:**
- How duplicate detections (same object detected in multiple tiles) are merged
- **NMS (Non-Maximum Suppression):** Standard approach — keeps highest confidence, discards overlapping boxes. Problem: in dense scenes (C2A has 35 objects/image), NMS can accidentally discard valid detections that happen to overlap
- **GREEDYNMM (Greedy Non-Max Merging):** Instead of discarding, it MERGES overlapping boxes. Better for dense scenes like C2A

**postprocess_match_metric=IOS:**
- How overlap between two boxes is measured for duplicate removal
- **IoU (Intersection over Union):** Standard. But when one box is much smaller than another (e.g., 8px tile detection vs 20px full-image detection of same person), IoU can be very low even for the same object
- **IOS (Intersection over Smaller):** Measures overlap relative to the SMALLER box. A 8px box inside a 20px box gets high IOS even though IoU would be low. Better for C2A's extreme size variation

### SAHI Results — What the Numbers Mean

| Config | VT Recall | F1 | Latency | Interpretation |
|--------|-----------|-----|---------|---------------|
| No SAHI | 0.767 | 0.850 | 45ms | Standard inference misses 23.3% of very-tiny people |
| SAHI-256 | **0.829** | 0.829 | 534ms | Catches 6.2% more tiny people, but precision drops (more false alarms) |
| SAHI-512 | 0.808 | **0.866** | 187ms | Best balance — catches 4.1% more tiny people while keeping precision high |

**Why SAHI-256 has lower F1 despite higher recall:**
- Tiny 256x256 tiles with conf=0.15 generate many false positives (P drops from 0.855 to 0.790)
- F1 = harmonic mean of P and R — the precision drop hurts F1
- But **F2** (recall-weighted) improves: 0.847 → 0.854, because in SAR, missing a person is worse than a false alarm

## TTA (Test-Time Augmentation)

### The Problem TTA Solves
A person lying down looks different than standing up. A person at image edge is partially visible. The model might detect them at one scale but miss them at another.

### How TTA Works — Step by Step

**Without TTA:**
```
Image (640x640) → Model → Detections
```
One pass, one chance.

**With TTA (augment=True in ultralytics):**
```
Image → Create 3 versions:
  1. Original scale (1.0×) → Model → Detections A
  2. Smaller scale (0.83×) → Model → Detections B  
  3. Even smaller (0.67×) → Model → Detections C
  Also: horizontal flip versions of each
  
→ Merge all detections (A + B + C + flipped versions)
→ Apply NMS to remove duplicates
→ Final detections
```

The model sees the same image at multiple scales and orientations. If it misses a person at one scale, it might catch them at another.

### TTA Resolution Parameter (imgsz)

When you set `imgsz=1280` with TTA:
- The 1.0x pass runs at 1280px (larger than training resolution 640px — objects appear bigger)
- The 0.83x pass runs at 1062px
- The 0.67x pass runs at 857px

A 6px person in a 640px image becomes ~12px in a 1280px image — much easier to detect!

### Why imgsz=1280 Works But 1920 Doesn't

| imgsz | 0.67x scale | 1.0x scale | 1.5x scale | Result |
|-------|------------|------------|------------|--------|
| 640 | 429px | 640px | 960px | Baseline — no extra info |
| 832 | 557px | 832px | 1248px | Moderate gain (+3.25% mAP50-95) |
| **1280** | **857px** | **1280px** | **1920px** | **Best (+5.39% mAP50-95)** |
| 1920 | 1286px | 1920px | 2880px | Degraded (-model confused) |

The model was **trained at 640px**. At 1280, the 0.67x scale (857px) is still close to training resolution. But at 1920, the 1.0x scale (1920px) is 3x the training resolution — the model sees feature patterns it was never trained on. It starts hallucinating.

### TTA Results — What the Numbers Mean

| Config | mAP50 | mAP50-95 | Interpretation |
|--------|-------|----------|---------------|
| No TTA (640) | 0.874 | 0.644 | Single pass, single scale |
| TTA@1280 | **0.890** | **0.698** | Multi-scale catches +5.4% more objects at tight IoU thresholds |

**Why mAP50-95 improves more than mAP50:**
- mAP50 measures detection at IoU=0.50 (loose — box roughly in the right place)
- mAP50-95 averages across IoU=0.50, 0.55, 0.60, ..., 0.95 (strict — box must be precise)
- TTA's multi-scale passes help the model produce more precise bounding boxes (not just detect, but localize accurately), so the gain is bigger at stricter IoU thresholds

## SAHI + TTA Combined

We tried running TTA inside each SAHI slice. Result: **identical to SAHI alone**.

**Why:** SAHI already slices to 256x256. TTA at slice level means running the model at 256×0.67=172px, 256×1.0=256px, 256×1.5=384px. All of these are BELOW the 640px training resolution. There's no benefit to multi-scale when all scales are smaller than what the model learned.

TTA needs to go ABOVE training resolution to help. SAHI slices are already below. They cancel each other out.

# PART 3: METRICS EXPLAINED

## Detection Metrics

### mAP50 (Mean Average Precision at IoU=0.50)
- **What it measures:** How well the model detects objects, where a detection counts as correct if it overlaps the ground truth box by at least 50%
- **How it's computed:** For each confidence threshold (0.01 to 0.99), compute precision and recall. Plot the Precision-Recall curve. The area under this curve = AP. Average across all classes = mAP. Since C2A has 1 class (person), AP = mAP.
- **IoU=0.50 means:** A predicted box is "correct" if it overlaps at least 50% with a ground truth box (measured by Intersection over Union)
- **Your number:** 0.890 (TTA@1280) — 89% of the area under the PR curve
- **Range:** 0 to 1 (higher is better)

### mAP50-95 (Mean Average Precision at IoU=0.50:0.05:0.95)
- **What it measures:** Same as mAP50, but averaged over 10 IoU thresholds: 0.50, 0.55, 0.60, 0.65, 0.70, 0.75, 0.80, 0.85, 0.90, 0.95
- **Why it matters:** Much stricter than mAP50. At IoU=0.95, the predicted box must almost perfectly match the ground truth. This measures both detection AND localization quality
- **Your number:** 0.698 (TTA@1280)
- **Always lower than mAP50** because higher IoU thresholds are harder to satisfy
- **This is THE primary metric** in modern object detection papers (COCO benchmark uses this)

### Precision (P)
- **What:** Of all boxes the model predicted, what fraction were actually correct?
- **Formula:** P = TP / (TP + FP)
- **TP** = True Positive (model predicted a box, and there IS a person there)
- **FP** = False Positive (model predicted a box, but there is NO person there — false alarm)
- **Your number:** 0.855 (no-aug baseline) — 85.5% of predictions are correct
- **High precision = few false alarms**

### Recall (R)
- **What:** Of all actual persons in the image, what fraction did the model find?
- **Formula:** R = TP / (TP + FN)
- **FN** = False Negative (there IS a person but the model MISSED them)
- **Your number:** 0.845 (no-aug baseline) — model finds 84.5% of all people
- **High recall = few missed people**
- **For search and rescue, recall is more important than precision** — missing a trapped person (FN) is worse than a false alarm (FP)

### F1 Score
- **What:** Harmonic mean of Precision and Recall — a single number that balances both
- **Formula:** F1 = 2 * P * R / (P + R)
- **Your number:** 0.850 (no-aug baseline)
- **If P=R, then F1=P=R. If one is much lower, F1 is pulled down**

### F2 Score
- **What:** Weighted harmonic mean that gives **4x more weight to recall** than precision
- **Formula:** F2 = 5 * P * R / (4 * P + R)
- **Why F2?** In disaster SAR, recall matters more. F2 captures this priority.
- **Your number:** 0.847 (no-aug baseline), 0.854 (SAHI-256) — SAHI improves F2 even though F1 drops, because recall improved significantly

### ECE (Expected Calibration Error)
- **What:** Measures how well the model's confidence scores match reality
- **Example:** If the model says "90% confident this is a person" for 100 predictions, roughly 90 should be correct. If only 70 are correct, the model is overconfident → high ECE
- **Formula:** Bins predictions by confidence (0-0.1, 0.1-0.2, ..., 0.9-1.0), computes |accuracy - confidence| for each bin, averages weighted by bin size
- **Your number:** 0.135 (no-aug), 0.146 (copy-paste) — lower is better
- **Copy-paste made calibration worse** — the model became less reliable in its confidence scores

## Size-Specific Metrics

### Very Tiny Recall (VT Recall)
- **What:** Recall computed only for objects with bounding box area < 64 pixels^2 (i.e., smaller than 8x8 pixels)
- **Your numbers:** 0.767 (baseline) → 0.829 (SAHI-256) — SAHI finds 6.2% more of these ultra-small people
- **C2A context:** 25,072 objects (34.6% of test set) are very tiny. This is the hardest category

### Size Categories Used

|----------|-----------|---------------|-------------------|------------|

| Tiny | 8^2 to 16^2 = 64-255 px^2 | 12x12 person | 20,520 | 28.3% |
| Small | 16^2 to 32^2 = 256-1023 px^2 | 24x24 person | 26,614 | 36.7% |

**Key takeaway:** 99.6% of all objects in C2A are smaller than 32x32 pixels. There are ZERO large objects. This is an extreme small-object detection problem.

# PART 4: TRAINING CONCEPTS

## Augmentation Parameters

### Copy-Paste Augmentation (copy_paste=0.5)
- **What:** During training, randomly select objects (people) from other training images, cut them out, and paste them into the current training image
- **Why:** If the model sees more small objects during training, it should learn to detect them better
- **copy_paste=0.5 means:** 50% probability of applying this to each training image
- **What went wrong:** C2A already has ~35 people per image. Adding more created unrealistically crowded scenes. The model got confused and forgot how to detect medium-sized people

### Mixup (mixup=0.15)
- **What:** Blend two training images together (alpha-blend). The model must detect objects from both images simultaneously
- **mixup=0.15 means:** 15% probability of applying this

### Mosaic (mosaic=1.0)
- **What:** Combine 4 training images into a single 2x2 grid. The model sees 4 different scenes at once
- **mosaic=1.0 means:** Applied to 100% of training images
- **close_mosaic=10 means:** Turn off mosaic for the last 10 epochs to let the model fine-tune on clean images

### Flipud (flipud=0.5)
- **What:** Vertical flip (upside down). Valid for aerial/nadir imagery since the camera looks straight down
- **Not used in no-aug model** (flipud=0.0)

### Degrees (degrees=10.0)
- **What:** Random rotation up to +-10 degrees
- **Not used in no-aug model** (degrees=0.0)

### Erasing (erasing=0.4)
- **What:** Randomly erase rectangular patches of the image during training. Forces the model to detect objects even with partial visibility (occluded objects)
- **erasing=0.4 means:** 40% probability

---

## Training Infrastructure

### DDP (Distributed Data Parallel)
- **What:** Training on multiple GPUs simultaneously. Each GPU processes a different batch of images, gradients are averaged
- **Your setup:** 2x T4 GPUs on Kaggle, batch=8 per GPU, effective batch=16

### AMP (Automatic Mixed Precision)
- **What:** Uses FP16 (half precision) for forward pass to save memory and speed up training, FP32 for loss computation to maintain accuracy
- **Effect:** ~40% faster training, ~30% less GPU memory

### Patience (patience=15)
- **What:** Early stopping — if mAP doesn't improve for 15 consecutive epochs, stop training
- **Your copy-paste model:** Trained all 120 epochs (never triggered early stopping)

# PART 7: ONE-LINE SUMMARY OF EACH EXPERIMENT

1. **Copy-Paste Augmentation:** Retrained the model with aggressive small-object pasting → FAILED (-1.67% mAP50-95, -13.25% medium recall). Good negative result for the paper.

2. **SAHI (256x256 slices):** Sliced images into small tiles so tiny people get upscaled and detected → **+6.24% very-tiny recall** (0.767 → 0.829). Best for finding the smallest people. Precision drops.

3. **SAHI (512x512 slices):** Larger tiles, less upscaling but better balanced → **+1.6% F1** (0.850 → 0.866). Best overall balance of precision and recall.

4. **TTA at 1280px:** Ran inference at 1280px with multi-scale → **+5.39% mAP50-95** (0.644 → 0.698). Beats YOLOv9-e (0.688). Best overall accuracy.

5. **SAHI + TTA combined:** Tried both together → **No additional gain**. TTA doesn't help when SAHI slices are already small.

6. **TTA at 1920px:** Pushed resolution too far → **Degraded**. Model can't handle 3x its training resolution.
