- **mAP50: 0.877 -> 0.890 (+1.6%)**
- **mAP50-95: 0.644 -> 0.698 (+5.4%)**
- This is a 5.4-point mAP50-95 improvement with ZERO retraining
- Confusion matrix at val3 (TTA 1280): 90% person recall, 10% miss rate — the best of all configs
- F1-confidence curve peaks at 0.85 F1 at conf=0.579 — healthy, well-calibrated

### 2. SAHI 256x256 is your best very-tiny recall — +6.2 points
- **Very-tiny recall: 0.767 -> 0.829 (+6.24%)**
- **Small recall: 0.896 -> 0.923 (+2.7%)**
- **Medium recall: 0.896 -> 0.896 (preserved!)** — the perform_standard_pred=True fix worked perfectly. Previous run lost 13.3% medium recall; this time zero loss.
- Overall recall: 0.845 -> 0.872 (+2.7%)
- **BUT precision dropped: 0.855 -> 0.790 (-6.5%)** — the conf=0.15 threshold is too aggressive. More FPs.
- F1 actually dropped: 0.850 -> 0.829 (-2.1%)

### 3. SAHI 512x512 is the sweet spot for balanced performance
- Highest F1 of all SAHI configs: **0.8657** (+1.6% over baseline)
- Very-tiny recall: 0.808 (+4.1%)
- Precision held at 0.867 (only -1.2% from baseline)
- Reasonable latency: 187ms
- **This is your best SAHI config for the paper if you need balanced P/R**

### 4. SAHI+TTA combined DID NOT help beyond SAHI alone
- SAHI(256)+TTA: identical metrics to SAHI(256) alone
- VT recall: 0.8292 (same), P/R/F1/F2: all identical
- Latency: 530ms (similar to SAHI alone at 534ms)
- **The TTA patch likely did not activate.** The monkey-patch wraps `model.__call__`, but SAHI may call the model differently internally. Or TTA's multi-scale passes at slice level (256px slices) don't add value because the slices are already small.
- This is not a script bug — it's a genuine finding: TTA adds nothing when slices are already small enough that multi-scale doesn't matter.

### 5. TTA at 1920 degraded vs 1280
- mAP50: 0.866 (vs 0.890 at 1280) — **WORSE**
- mAP50-95: 0.682 (vs 0.698 at 1280) — worse
- This makes sense: at 1920, the model processes far beyond its training resolution (640). The 0.67x TTA scale runs at 1286px which is good, but the 1.0x scale at 1920px sees feature distributions it was never trained on.
- **1280 is the sweet spot. Don't go higher.**

### 6. Confusion matrices are consistent and healthy

- val2 (TTA 832): 90% recall, 10% miss

## WHAT'S WRONG / CAVEATS

### 1. SAHI mAP50/mAP50-95 are not reported
- SAHI uses custom per-image TP/FP/FN matching, not ultralytics' official mAP calculation
- The mAP50/mAP50-95 columns show "-" for SAHI configs
- For the paper, you'll need to clearly state: "SAHI metrics computed via per-image box matching at IoU=0.5" vs "mAP computed via ultralytics val()"
- This is standard practice — SAHI papers all use custom evaluation

### 2. SAHI 256x256 precision is low (0.790)
- conf=0.15 with 256x256 slices generates many false positives
- F1 actually dropped vs baseline (0.829 vs 0.850)
- **For SAR applications this is acceptable** — missing a human (FN) is worse than a false alarm (FP)
- F2 score (recall-weighted) IMPROVED: 0.847 -> 0.854
- **In the paper, argue via F2: "In disaster SAR, recall matters more than precision"**

### 3. SAHI+TTA combined is a non-result
- Identical to SAHI alone — the TTA patch likely didn't activate or didn't matter
- **Don't report this as a separate config in the paper** — just report SAHI and TTA independently
- If reviewers ask "did you try combining them?" — say yes, it showed no additional gain
