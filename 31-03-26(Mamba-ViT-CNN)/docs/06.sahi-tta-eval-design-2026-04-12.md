# SAHI + TTA Evaluation Script Design — April 12, 2026

## What This Script Does

Pure evaluation (NO training) of the original no-aug Mamba+CBAM+P2 model using
two inference-time enhancement techniques:

1. **SAHI** (Slicing Aided Hyper Inference) — tiles images, detects per tile, merges
2. **TTA** (Test-Time Augmentation) — multi-scale + flip inference
3. **SAHI + TTA combined** — patches SAHI to use TTA per slice

## Key Improvements Over Previous Run's SAHI Cell

| Issue | Previous Run | This Script | Why |
|-------|-------------|-------------|-----|
| Postprocess | NMS | **GREEDYNMM** | Merges overlapping detections instead of discarding — critical for dense C2A (20-40 objects/image) |
| Match metric | IoU (implicit) | **IOS** | Intersection-over-Smaller handles size-varying duplicates from adjacent slices |
| Full-image pred | Not used | **perform_standard_pred=True** | Preserves medium/large recall (previous run lost -13.3% medium recall) |
| Slice sizes | 512, 640, 768 | **256, 320, 512, 640** | C2A has 47% sub-10px objects. 320x320 (0.5x training) is literature-recommended |
| Confidence | 0.25 | **0.15-0.20** | Sub-10px objects produce lower confidence; let GREEDYNMM handle FPs |
| TTA imgsz | 832, 1024 | **832, 1280, 1920** | VisDrone benchmarks: +9 mAP@50 from 640->1280. 1280 is the sweet spot |
| SAHI+TTA combo | Not done | **Monkey-patched** | Wraps YOLO.__call__ to inject augment=True per slice |

## Potential Risks and Mitigations

### 1. Custom model loading with SAHI
**Risk**: SAHI does `YOLO(model_path)` internally, which needs C3K2Mamba importable.
**Mitigation**: All custom classes (C3K2Mamba, LocalWindowSSM, _MambaBottleneck, _SelectiveScan1D, CBAM) are defined at module level AND registered via setattr on ultralytics.nn.tasks/modules before any model loading.

### 2. SAHI+TTA monkey-patch correctness
**Risk**: Naively rewriting perform_inference breaks SAHI's internal state tracking.
**Mitigation**: Instead of rewriting, we wrap YOLO.__call__ at instance level to inject augment=True, then delegate to SAHI's original perform_inference. This preserves shape tracking, result parsing, and config passing.

### 3. Memory on T4 with large imgsz
**Risk**: TTA at 1280/1920 with batch>1 may OOM.
**Mitigation**: batch=1 for 1280/1920, try/except on OOM with graceful skip.

### 4. SAHI latency
**Risk**: 256x256 slicing on 640x640 images creates many tiles (~20-25), each processed individually.
**Mitigation**: This is expected. SAHI is not real-time. For SAR applications, 200-500ms/image is acceptable. Latency is reported in results.

### 5. FP explosion with low confidence
**Risk**: conf=0.15 may produce many false positives.
**Mitigation**: GREEDYNMM with IOS metric at threshold 0.5 filters duplicates. Per-size recall metrics will show if precision drops unacceptably.

## How to Run

1. Upload to Kaggle: C2A dataset + best.pt (no-aug) + yolo11m.pt
2. Paste script into notebook, set GPU T4 x2, Internet ON
3. TEST_MODE=True first (smoke test, ~5 min)
4. TEST_MODE=False for full run (~2-3 hours)
5. Download sahi_tta_noaug_results.zip from /kaggle/working/
