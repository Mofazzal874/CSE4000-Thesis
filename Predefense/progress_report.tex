\chapter{Introduction}

\section{Overview}
This summary report examines three research papers in the field of real-time human detection in disaster scenarios using drone imagery. The first paper focuses on deep learning approaches for human detection from aerial images in search and rescue missions, emphasizing the problems in real-world search and rescue missions with varying lighting conditions. The second paper presents a multi-object tracking system for moving UAV platforms, addressing challenges of rapid viewpoint changes and camera motion. The third paper introduces TPH-YOLOv5, an improved object detection framework that integrates transformer prediction heads to enhance performance with small and densely packed objects.

\section{Problem Statement}
Our primary goal is to achieve \textbf{human detection in disaster scenarios using aerial devices like drones.}. The main challenges include:

\begin{itemize}
\item \textbf{Scale Variations:} Humans appear as extremely small objects in aerial imagery.
\item \textbf{Occlusion and Partial Visibility and Dense Object:} Humans are occluded by debris, buildings, or structures and can appear multiple in close proximity.
\item \textbf{Environmental Conditions:} Varying light conditions, weather effects and smoke.
% \item \textbf{Real-time Processing Requirements:} Emergency response scenarios demand immediate results, constraining computational complexity and processing time
\item \textbf{UAV Motion Effects:} Camera shake, rapid viewpoint changes,  motion blur etc.
\item \textbf{Class Imbalance:} The vast background creates  imbalance between human objects and background regions
\end{itemize}

\section{Paper 1: Deep Learning for Human Detection from Aerial Images in Search and Rescue Missions}

\subsection{Overview}
This paper denotes  the  challenges of detecting humans from aerial imagery during Search and Rescue (SAR) operations. It focuses on improving detection performance under varying lighting conditions—a significant limitation in existing models and datasets. The authors augmented the SARD (SAR Dataset) to include diverse lighting scenarios and evaluated six YOLO model variations to identify the most effective solution for real-world SAR missions.

\subsection{Methodology}

\subsubsection{Data Collection \& Augmentation}
The base SARD dataset has 1,981 manually annotated images from video frames. Images depicted people in various states (exhaustion, injury, sitting and walking ) . For the augmentation part , The authors applied brightness and contrast adjustment to simulate like real-world's varying light conditions and weather conditions. The final dataset contained 4,000 images

The dataset was divided as:
\begin{itemize}
\item Training: 3,000 images (75\%)
\item Validation: 600 images (15\%)
\item Testing: 400 images (10\%)
\end{itemize}

Six detectors, all fine-tuned:
\[
\text{YOLOv8n, v8s, v8m, \; YOLOv10n, v10s, v10m.}
\]

\section{Paper 2: Multi-Object Tracking Meets Moving UAV}

\subsection{Overview}
This paper present UAVMOT, a one-stage Multi-Object Tracking (MOT) system  for moving UAV videos. The research addresses the unique challenges of object tracking from drone perspectives, including rapid viewpoint changes, camera motion changes, and the challenge small objects in aerial footage. The system augments a CenterNet/FairMOT-style detector with three specialized components designed to handle UAV -specific tracking effectively.

\subsection{Methodology}

\subsubsection{System Architecture}

\paragraph{Backbone:}
The UAVMOT system is built on a DLA-34 backbone feature extractor, with shared features feeding three parallel heads in a CenterNet-style architecture:
\begin{itemize}
  \item \textbf{Heatmap} ($\mathbf{H_m}$): object-center likelihood, one channel per class.
  \item \textbf{Width/Height (wh)}: regresses bounding-box size at each center.
  \item \textbf{FID}: ReID (identity) embedding map used for association.
\end{itemize}

\subsubsection{Key Components}

\textbf{IDFU (ID Feature Update):} Uses adjacent frames(t and t-1 th frame) to adapt the identity embeddings under UAV view change. Concretely: (i) extract \emph{top-$K$} ID features from the previous frame, compress them; (ii) compute cross-frame correlation to obtain attention weights; (iii) fuse the attention-weighted previous features with current-frame ID features via conv to update the embeddings used for tracking.

\medskip
\textbf{AMF (Adaptive Motion Filter):} Switches association by UAV motion mode. When motion is \emph{normal}, use IoU+Kalman; when \emph{abnormal} (e.g., turns/accel), use a \emph{local relation filter} that builds a relative-geometry vector $v=[\theta,l_{\max},l_{\min}]$ to neighbors and forms a position-similarity matrix, which is fused with ID similarity before matching those frames.

\medskip
\textbf{GBF (Gradient-Balanced Focal) loss:} Supervises the heatmap head with reweighting that (i) balances long-tailed categories and (ii) emphasizes small objects. Implemented as category-balance weights $W_b$ and small-object weights $W_s$ applied to the heatmap cross-entropy: $\mathrm{GBF}=W_b \cdot W_s \cdot L_{\mathrm{Hm}}$.

\subsection{Results}

\subsubsection{Benchmark Performance}
The system achieved state-of-the-art results on standard UAV tracking datasets:

\section{Paper 3: TPH-YOLOv5 — Improved YOLOv5 Based on Transformer Prediction Head for Object Detection on Drone-Captured Scenarios}

\subsection{Overview}
TPH-YOLOv5 targets object detection in drone (UAV) imagery.The research extends YOLOv5 with an extra tiny-object head, replaces prediction heads of YOLOv5 with Transformer Prediction Heads (TPH), and integrats CBAM attention mechanisms.The implemeted system achieved state-of-the-art competitive results on VisDrone2021 with 39.18\% AP (+1.81 over prior SOTA and approximately 7\% improvement over YOLOv5 baseline).

\subsection{Methodology}

\subsection{Data Augmentation Steps}
\begin{itemize}
    \item Photometric Distortion
    \item Geometric distortion(scaling , cropping , translation, shearing, rotating)
    \item MixUp, Mosaic, and copy-paste augmentation for tiny or partially occluded human instances.
\end{itemize}

\subsubsection{System Architecture}
Architecture: 
  \begin{itemize}
      \item baseline: YOLOv5x
      \item Backbone:CSPDarknet53 ( as the feature extractor) 
      \item Neck: PANet (Path Aggregation Network) 
  \end{itemize}

\subsubsection{Key Architectural Improvements}

\textbf{Extra Tiny-Object Head (P2):}
\begin{itemize}
\item Added a fourth detection head utilizing high-resolution features specifically to better detect very small instances.
\end{itemize}

\textbf{Transformer Prediction Heads (TPH):}
\begin{itemize}
\item Replaced standard prediction heads with transformer encoder blocks
\item Implemented self-attention mechanisms at head and end of backbone
\item Reduces GFLOPs while improving performance
\end{itemize}

\textbf{CBAM Attention Integration:}
\begin{itemize}
\item Inserted Convolutional Block Attention Module (CBAM)
\item Helps resist background clutter common in aerial imagery
\end{itemize}

\chapter{My Observations}

\section{Paper 1: Deep Learning for Human Detection from Aerial Images in Search and Rescue Missions}

\begin{itemize}
\item \textbf{Dataset Limitations:} Though the augmented dataset (4,000 images) addresses lighting variations, it may still be insufficient for real-world deployment. The original SARD dataset's wilderness focus might not adequately represent different disaster scenarios like flood , traffic accident, fire or urban search and rescue. 

\item \textbf{Environmental Robustness:} Although lighting conditions are addressed, other critical factors like weather conditions (rain, fog, smoke), motion blur from UAV movement, and varying altitudes are not evaluated.

\item \textbf{Evaluation Scope:} The comparison focuses mainly on YOLO variants but lacks evaluation against other state-of-the-art detection frameworks that might be more suitable for aerial imagery.

\end{itemize}

\section{Paper 2: Multi-Object Tracking Meets Moving UAV}

\begin{itemize}
\item \textbf{Real-time Performance Trade-offs:} The computational overhead of IDFU and AMF  could be problematic for mobile UAV platforms.
\item \textbf{Low FPS:} While achieving 12 FPS with DLA-34, it is not very suitable for even moderate speed drones.

\item \textbf{Environmental Adversary:} The system's performance under adverse weather conditions, varying lighting, or when tracking partially occluded objects (common in debris-filled disaster zones) requires further validation.

\end{itemize}

\section{Paper 3: TPH-YOLOv5 — Improved YOLOv5 Based on Transformer Prediction Head for Object Detection on Drone-Captured Scenarios}

\begin{itemize}
\item \textbf{Computational and Memory Constraints:} The added P2 head significantly increases parameters and forces batch size limitations (batch size 2). This creates problem for mobile UAV deployments.

\item \textbf{Inference Latency Trade-offs:} Multi-scale testing with 6 views and Weighted Boxes Fusion ensembling improve accuracy but introduce notable inference latency.

\item \textbf{Dataset Specificity Concerns:} The evaluation focuses primarily on VisDrone dataset. Cross-dataset generalization to other UAV domains, particularly disaster scenarios with different object classes and environmental conditions, remains unverified and could limit practical applicability.

\end{itemize}

\chapter{Research Gaps \& Proposed Methods}

\section{Research Gaps in the Discussed Papers}

The comprehensive analysis of the three papers reveals several critical research gaps that limit their practical applicability in real-world disaster scenarios:

\subsection{Cross-Domain Generalization}
\begin{itemize}
\item \textbf{Dataset Specificity:} All three papers focus on specific datasets (SARD, VisDrone, UAVDT) to detect humans but in Disaster scenarios , They may not perform bare minimum. 
\item \textbf{Environmental Adversity:} Though all of them worked on adverse condition ,Limited evaluation under extreme weather conditions, smoke-filled environments, or low-visibility scenarios commonly encountered in disaster zones, which may create problem.
\item \textbf{Terrain Variation:} Insufficient consideration of varying geographical terrains (urban, rural, coastal, mountainous) that affect detection and tracking performance
\end{itemize}

\subsection{Real-time Deployment Challenges}
\begin{itemize}
\item \textbf{Inference Speed vs. Accuracy Trade-offs:} Inadequate exploration of lightweight architectures.
\item \textbf{Edge Computing Optimization:} Lack of optimization for edge devices, mobile devices like drones .
\end{itemize}

\subsection{Multi-modal Integration}
\begin{itemize}
\item \textbf{Thermal-RGB Fusion:} Limited exploration of thermal imaging integration, which is crucial for detecting humans in smoke, darkness, or debris-covered areas
\end{itemize}


\section{Proposed Methods}

Based on the identified research gaps, we propose several methodological improvements and approachs:

\subsection{  Attention heads/filter optimized for Disaster scenarios}
\begin{itemize}
\item \textbf{New Attention heads mechanism:} New attention head mechanism that will improve performance for disaster specific scenarios.
\item \textbf{Two-stage filtering:} Instead of Single-stage filtering , we can work with Two-stage filtering with a small trade-off with detection speed.
\end{itemize}

\subsection{Lightweight Real-time Architecture}
\begin{itemize}
\item \textbf{Knowledge Distillation Pipeline:} Develop teacher-student frameworks to compress complex models while preserving critical detection capabilities
\end{itemize}

\subsection{Disaster-specific Data Augmentation}
\begin{itemize}
\item \textbf{Synthetic Disaster Scenarios:} Generate realistic disaster environments using advanced simulation techniques for training data augmentation
\end{itemize}

\subsection{Integrated Operational Framework}
\begin{itemize}
\item \textbf{Real-time Communication Protocol:} Develop efficient protocols for transmitting detection results to ground control with priority-based filtering so that we can get rid of edge device computation problem.
\end{itemize}
\subsection{Proposed Sytem: Hierarchical Body-Part Attention (HBPA) Mechanism}
\begin{itemize}
\item \textbf{Priority-based detection:} Priority-based detection with head/neck getting 60\% attention weight
\item \textbf{Body Part Decomposition Module:} Body Part Decomposition Module to split body into three major region
\item \textbf{Convolutional Block Attention Model (CBAM):} Convolutional block attention model(CBAM) to find attention region on scenarios with dense objects.
\end{itemize}
